# Тема 1: Основы памяти и адресов 🧠

## 📖 Теория

### 🧠 Что такое оперативная память?

**Оперативная память (RAM)** — это физическое устройство, которое используется компьютером для временного хранения данных, необходимых для выполнения программ.

#### Ключевые понятия:
- Память состоит из **миллиардов ячеек**.
- Каждая ячейка имеет **уникальный адрес** (как номер дома на улице).
- Обычно каждая ячейка хранит **1 байт** (8 бит) данных.
- Адресация позволяет процессору читать и записывать данные по конкретному адресу.

> Аналогия: Представьте память как **огромный ряд почтовых ящиков**. У каждого ящика есть номер (адрес), и в каждом ящике можно хранить какое-то значение (данные).

---

### 📦 Что такое переменная?

**Переменная** — это **именованная область памяти**, которая хранит данные определённого типа.

#### Характеристики переменной:
- **Имя** — идентификатор, который использует программист.
- **Тип** — определяет, сколько памяти занимает переменная и как интерпретировать данные.
- **Адрес** — положение переменной в памяти.
- **Значение** — содержимое области памяти.

#### Пример:
```cpp
int x = 42;
```
- `int` — тип (обычно 4 байта)
- `x` — имя
- Где-то в памяти выделено 4 байта со значением 42
- У этих 4 байтов есть начальный адрес

> Важно: переменная — это не просто значение, а **область памяти + имя**.

---

### 🔍 Оператор взятия адреса `&`

Оператор **`&`** (амперсанд), применённый к переменной, возвращает её **адрес в памяти**.

#### Синтаксис:
```cpp
тип переменная = значение;
cout << &переменная;
```

---

### 📝 Пример 1: Базовое получение адресов переменных

Файл: `01_address_of_variable.cpp`

```cpp
#include <iostream>

int main() {
    int a = 10;
    double b = 3.14;
    char c = 'Z';
    
    std::cout << "=== Адреса переменных ===" << std::endl;
    std::cout << "a (int):    значение = " << a << ", адрес = " << &a << std::endl;
    std::cout << "b (double): значение = " << b << ", адрес = " << &b << std::endl;
    std::cout << "c (char):   значение = " << c << ", адрес = " << static_cast<void*>(&c) << std::endl;
    
    return 0;
}
```

#### Пояснения к примеру:
- Каждая переменная имеет уникальный адрес.
- Для `char` используется `static_cast<void*>(&c)`, чтобы избежать интерпретации как строки.
- Адреса выводятся в шестнадцатеричном формате (например, `0x7ffd8a3b4c1c`).

---

### 📐 Связь между адресом и значением

```cpp
int x = 5;
int* ptr = &x;   // сохраняем адрес x в указатель (будет в следующей теме)
```

- По адресу можно **найти значение**
- Адрес не зависит от значения, значение может меняться, адрес остаётся тем же

```cpp
x = 10;          // значение изменилось
// адрес &x остался тем же
```

### 📊 Пример 2: Расположение переменных в памяти

Файл: `02_memory_layout.cpp`

```cpp
#include <iostream>

int global_var = 42;        // глобальная переменная
static int static_var = 100; // статическая переменная

int main() {
    int local_var = 10;      // локальная переменная
    int* heap_var = new int(5); // переменная в куче
    
    std::cout << "=== Расположение в памяти ===" << std::endl;
    std::cout << "Код функции main:      " << reinterpret_cast<void*>(&main) << std::endl;
    std::cout << "Глобальная переменная: " << &global_var << std::endl;
    std::cout << "Статическая переменная: " << &static_var << std::endl;
    std::cout << "Локальная переменная:  " << &local_var << std::endl;
    std::cout << "Переменная в куче:     " << heap_var << std::endl;
    
    delete heap_var;
    return 0;
}
```

#### Что демонстрирует этот пример:
- **Глобальные и статические переменные** располагаются в сегменте данных.
- **Локальные переменные** — в стеке.
- **Динамическая память** — в куче.
- **Код функции** — в сегменте кода.
- Адреса разных сегментов существенно отличаются.

### 🔬 Сравнение адресов разных переменных

Файл: `03_address_comparison.cpp`

```cpp
#include <iostream>

int main() {
    int x = 5;
    int y = 10;
    int z = 15;
    
    std::cout << "=== Сравнение адресов ===" << std::endl;
    std::cout << "x: " << &x << std::endl;
    std::cout << "y: " << &y << std::endl;
    std::cout << "z: " << &z << std::endl;
    
    // Адреса идут подряд или с небольшим смещением
    std::cout << "\nРазница между &x и &y: " 
              << reinterpret_cast<char*>(&y) - reinterpret_cast<char*>(&x) << " байт" << std::endl;
    std::cout << "Разница между &y и &z: " 
              << reinterpret_cast<char*>(&z) - reinterpret_cast<char*>(&y) << " байт" << std::endl;
    
    // Переменные разных типов
    int a = 100;
    double b = 200.5;
    char c = 'C';
    
    std::cout << "\nПеременные разных типов:" << std::endl;
    std::cout << "a (int):    " << &a << std::endl;
    std::cout << "b (double): " << &b << std::endl;
    std::cout << "c (char):   " << static_cast<void*>(&c) << std::endl;
    
    return 0;
}
```

#### Наблюдения:
- Локальные переменные часто располагаются **рядом** в стеке.
- Разница в адресах обычно соответствует размеру типа (или больше из-за выравнивания).
- Переменные разных типов также могут располагаться с учётом выравнивания.

### ⚠️ Особенности вывода адресов типа `char`

Файл: `04_char_address.cpp`

```cpp
#include <iostream>

int main() {
    char ch = 'A';
    char str[] = "Hello";
    
    std::cout << "=== Проблема с char* ===" << std::endl;
    
    // Проблема: cout интерпретирует char* как строку
    std::cout << "Адрес ch (обычный вывод): " << &ch << std::endl;
    std::cout << "Адрес str (обычный вывод): " << str << std::endl;
    
    // Правильный способ: приведение к void*
    std::cout << "\nПравильный вывод:" << std::endl;
    std::cout << "Адрес ch: " << static_cast<void*>(&ch) << std::endl;
    std::cout << "Адрес str: " << static_cast<void*>(str) << std::endl;
    
    // Демонстрация проблемы с указателями
    char* ptr = &ch;
    std::cout << "\nУказатель на char:" << std::endl;
    std::cout << "ptr (как есть): " << ptr << std::endl;
    std::cout << "ptr (приведённый): " << static_cast<void*>(ptr) << std::endl;
    
    return 0;
}
```

#### Почему это важно:
- В C++ `std::cout` обрабатывает `char*` особым образом — как **C-строку** (выводит символы до `\0`).
- Это приводит к тому, что адрес переменной типа `char` выводится не как адрес, а как строка, начинающаяся с этого символа.
- **Решение**: явно привести к `void*`, чтобы получить именно адрес.
- Это одна из частых "ловушек" для новичков.

---

### 💡 Важные замечания

1. **Адрес переменной нельзя изменить** — он фиксирован на время жизни переменной.
2. **Локальные переменные** хранятся в стеке (stack), их адреса могут меняться при каждом вызове функции.
3. **Глобальные переменные** имеют фиксированные адреса на всё время работы программы.
4. **Шестнадцатеричный формат** — стандартный способ отображения адресов в C++ (префикс `0x`).
5. **Для `char` всегда используйте приведение к `void*`** при выводе адреса.

## ⚠️ Частые ошибки

```cpp
// ❌ Неправильно
char c = 'A';
cout << &c;  // Выведет "A" и дальше мусор!

// ✅ Правильно
cout << (void*)&c;  // Выведет адрес

// ❌ Неправильно
int &ref = &x;  // Путаница ссылки и адреса

// ✅ Правильно
int* ptr = &x;  // Указатель хранит адрес
```

## 📝 Заметки

- Адреса могут меняться при каждом запуске (ASLR - рандомизация адресного пространства)
- В 64-битных системах адреса занимают 8 байт
- Префикс `0x` означает шестнадцатеричное число
